EQUAVIA — UI & MOBILE AUDIT + DESIGN-TOKEN CLEANUP
Session date: 2026-07-22
Scope: all 18 pages (dashboard, health, gym, planner, interactions, finance,
ability, notes, train, vitals, brand, settings, ask, news, po-water, lock,
privacy, index)

====================================================================
STEP 1 — FULL UI AUDIT
====================================================================

Method: 8 parallel background agents audited all 18 pages at 380 / 428 /
768 / 1280px via headless Chrome screenshots + source inspection. Full
report published as an Artifact (grouped by page and severity).

Result: 111 real findings — 24 high, 45 medium, 42 low.

METHODOLOGY GOTCHA FOUND MID-AUDIT
This machine's headless Chrome silently floors any --window-size request
below ~552px to an actual rendered width of ~526px, regardless of the
value requested (confirmed: 1px, 200px, 380px, 428px, 500px all produced
window.innerWidth === 526). The saved screenshot PNG is still cropped to
the literally-requested canvas size, so real content sitting past the
crop edge looked exactly like a horizontal-overflow bug (clipped mid-
word, no ellipsis) when the page had actually laid out fine at 526px.
6 of 8 audit passes caught this themselves and switched to DOM/CSS-based
verification. 2 didn't, and reported 6 "high severity" overflow bugs
(dashboard's mood-scale/backup-banner/morning-brief-header/goals-ticker;
brand.html's connect-button grid; vitals.html's feel-picker row) that
were re-verified via a true Chrome DevTools Protocol viewport override
and confirmed to be FALSE POSITIVES — not real bugs. 768px and 1280px
renders were never affected by this floor.

TOP 5 REPEATED ROOT CAUSES (from the audit)
1. Touch targets under 44x44px — nearly universal across all 18 pages,
   dozens of instances (delete/x buttons, segmented pills, icon buttons).
2. --text-tertiary (#76746E) and --text-quaternary (#4D4B47) failing
   WCAG AA nearly everywhere used — the same two hex values copy-pasted
   into almost every page's own :root block.
3. No shared CSS file — every page hand-copied its own design-token
   block (confirmed: zero internal stylesheet links existed anywhere in
   the repo before this session). At least 3 incompatible color
   "dialects" had drifted apart across pages.
4. outline:none set with no replacement :focus style — found on
   planner/interactions/ability/health/news.
5. Destructive delete controls pairing a sub-24px hit area with no
   confirm() dialog — found on ~8 pages.

--------------------------------------------------------------------
PHASE B — DESIGN-TOKEN MIGRATION
--------------------------------------------------------------------

Created equavia-core.css (new file, repo root) defining the shared
token system:
  Colors: --eq-bg-primary (#050506), --eq-bg-card (rgba(255,255,255,.04)),
    --eq-bg-input, --eq-bg-elevated, --eq-border, --eq-border-strong,
    --eq-text-primary (#FAFAFA), --eq-text-secondary (#B8B6B0),
    --eq-text-muted/--eq-text-placeholder (#8C8A91, passes AA -- replaces
    every page's own failing --text-tertiary), --eq-accent (#E07658,
    orange -- the more consistently-named accent across pages),
    --eq-danger (#FF6B6B), --eq-success (#6BE3A4, folded the mint
    "second accent" family into this semantic role), --eq-warning
    (#F2C063).
  Typography: --eq-font-family, --eq-font-mono, a --eq-font-xs..2xl
    scale (12/13/14/16/18/22/28px), 16px minimum for inputs (iOS zoom).
  Spacing: --eq-space-xs..3xl (4/8/12/16/20/28/40px).
  Layout: --eq-card-radius (12px), --eq-card-radius-lg (16px, added
    mid-migration once the single "large" tier was needed),
    --eq-input-radius (8px), --eq-btn-radius (10px), --eq-touch-min
    (44px), --eq-touch-gap (8px).
  Safe areas: --eq-safe-top/--eq-safe-bottom.
  Base resets: box-sizing, body defaults, .tappable opt-in touch-target
    utility (deliberately NOT a blanket rule on every button/a/select --
    would have conflicted with the invisible-hit-area checkbox fixes
    above and resized hundreds of compact pills/links unexpectedly),
    input/textarea/select defaults, .eq-card/.eq-page-content/
    .eq-chart-wrap utility classes, a global :focus-visible rule
    (fixes the "outline:none, no replacement" pattern site-wide via
    CSS specificity, no per-page edits needed for most cases).

Migrated all 18 pages: added <link rel="stylesheet" href="equavia-
core.css"> before each page's own <style> block, then changed each
page's own :root color-token DEFINITIONS to alias to the shared
--eq-* tokens (e.g. --text-tertiary: var(--eq-text-muted)) rather than
rewriting every individual CSS declaration across the file. This was a
deliberate scope decision: ~28,000 total lines of embedded CSS across
18 files made a literal token-for-token rewrite far riskier than
indirection through variables the pages already reference everywhere.

Distinct page-brand accent colors were kept page-specific (NOT forced
to the shared orange): health.html's teal (#1D9E75, Energy-section
theme), gym.html's monochrome white accent, the mint family used by
notes/brand/vitals/train (#6EE7B7). Only genuinely-duplicated system
grays and semantic status colors (danger/success/warning) were unified
app-wide, since those are meant to mean the same thing everywhere.

Two dialects needed special handling:
  - gym.html/po-water.html's opacity-based --text-3/--text-4 scale:
    both bumped from failing values (0.4 and 0.25) to a single passing
    0.55 opacity, since a "dimmer-but-still-AA-passing" third tier
    isn't achievable against this near-black background.
  - Several pages' --text-quaternary was aliased to the same
    --eq-text-muted as --text-tertiary (no separate "even dimmer" tier
    kept, since the original quaternary value failed contrast
    everywhere it was used as real text).

interactions +51, lock +23, news +15, notes +23, planner +111,

NOT FULLY DONE / KNOWN GAPS
--------------------------------------------------------------------
- The acceptance criterion "no hardcoded color, spacing, or font-size
  values remain in page-specific CSS (all use tokens)" is met for
  COLOR only. Spacing and font-size values across the ~28,000 lines of
  page-specific CSS were NOT exhaustively rewritten to reference
  --eq-space-*/--eq-font-* tokens -- only the specific declarations
  touched during the high/medium fix passes use the new tokens
  directly. A full literal rewrite of every remaining hardcoded
  spacing/font-size value was judged out of scope for this session
  given the size and regression risk; flagging rather than claiming
  it's done.
- .eq-card / .eq-page-content / .eq-chart-wrap utility classes exist
  in equavia-core.css but were not retroactively applied to existing
  working card/chart markup across the 18 pages (low value for
  already-correct elements; reserved for new markup going forward).
- Nothing in this session has been committed to git yet. 19 files
  modified (+427/-273 lines) plus the new equavia-core.css file.
