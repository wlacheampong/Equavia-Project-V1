Privacy Policy

Last updated: July 12, 2026

Equavia ("this app") is a personal, single-user dashboard for tracking daily goals, health, fitness, water intake, and finances. It is not a commercial product, has no user accounts, and is not intended for use by the general public — it exists to serve its one operator. This policy explains what data the app touches and where it goes, including for anyone reviewing it as part of a third-party API integration (e.g. WHOOP's developer platform).

What data this app handles

Depending on which page you use, the app reads or stores:

  • Goals & to-do data — text of daily/tomorrow goals, completion status, streak counts.
  • Water & health inputs — hydration logs, and profile inputs you provide (weight, age, sex, activity level, caffeine/substance intake) used only to calculate a daily water target.
  • Fitness data — workout/progressive-overload entries you log.
  • Finance data — entries you add on the finance page.
  • WHOOP biometric data (only if you connect a WHOOP account) — recovery score, sleep stages/duration, strain, HRV, resting heart rate, skin temperature, blood oxygen, and respiratory rate, as provided by WHOOP's API.
  • Google Calendar data (only if you connect a Google account) — event titles, times, dates, and calendar names for events you view or create through the app, as provided by Google's Calendar API.

Where it's stored

All data is first stored locally in your browser (localStorage) on the device you're using. Some pages additionally sync that same data to a private Supabase (PostgreSQL) database so it stays consistent across your own devices. This app has no accounts or login — the Supabase project is used solely to mirror this one operator's data between their own devices, not to collect data from visitors.

Third-party services

  • Supabase — stores a mirror of the app data described above for cross-device sync. See Supabase's privacy policy.
  • WHOOP API — if you choose to connect a WHOOP account, the app requests an OAuth token from WHOOP and uses it (via a backend proxy the operator controls) to fetch the biometric data listed above. This app does not sell or share WHOOP data with any other party. See WHOOP's privacy policy.
  • Google Calendar API — if you choose to connect a Google account, the app requests an OAuth token from Google and uses it (via a backend proxy the operator controls) to read your calendar events and create new ones on your behalf. This app does not sell or share Google account data with any other party. See Google's privacy policy.
  • Anthropic (Claude API) — used in two places, both opt-in and both inactive unless an API key is configured server-side: (1) an optional "✨ Polish" feature that sends the text of a goal you're typing to Anthropic's API to reword it; (2) the "Ask Claude" page, which sends your typed message plus a compact text summary of whichever data domains (Goals/Tasks, Health, Fitness, Finance, Habits, Relationships) you've toggled on for that conversation — nothing is sent for a domain you haven't enabled, and the toggle is per-message, not sticky forever. Conversation history for the Ask Claude page is stored locally the same way as the rest of the app's data. See Anthropic's privacy policy.

Data sharing

Data is not sold, rented, or shared with advertisers or data brokers. It is only sent to the third-party services listed above, and only to provide the specific feature you're using (sync, WHOOP metrics, or goal polishing).

Your control over your data

  • Clearing your browser's site data removes everything stored locally.
  • Disconnecting WHOOP (via the "Disconnect" button) stops any further data access and removes the locally stored token.
  • Because there are no user accounts, data is scoped to whoever operates this app — there is no multi-user data to separate or export.

Children's privacy

This app is not directed at children and is not intended for public sign-up of any kind.

Changes to this policy

If what this app collects or how it's used changes, this page will be updated with a new "Last updated" date.
